### 🔍 [AI Image Model Comparison](https://1beebe.github.io/ai-agent-portfolio/image-model-comparison/)
**The enablement lesson:** "Which AI tool is best?" is the wrong question. Teams need to understand which tool serves their specific use case.

**What it does:**  
A MindStudio workflow that runs 6 image generation models in parallel against the same prompt and delivers a visual comparison - all from a single text input.

**Why it matters for enablement:**  
This teaches teams to evaluate AI tools based on their needs (brand consistency vs. technical accuracy vs. speed) rather than chasing "best practices."

**How it works:**  
User submits prompt → LLM enhances it → 6 models run simultaneously → alt text generated → custom frontend renders side-by-side gallery.

**Models compared:** Gemini 3 Pro · Flux Pro 2 · Kling Image O3 · Ideogram V3 · Stable Image Ultra · GPT Image 1.5

### 📄 [Document Processing & Archival Agent](https://github.com/1beebe/document-archivist)
**The enablement lesson:** AI can handle tedious manual tasks - but only if you understand the workflow logic.

**What it does:**  
Automatically processes PDF documents from email attachments, cleans OCR text, and organizes documents with structured metadata.

**Why it matters for enablement:**  
This demonstrates how to chain AI actions (email monitoring → PDF processing → text extraction → metadata tagging) - the foundation non-technical teams need to automate their own workflows.

**Use case:**  
Teams drowning in document management can see exactly how AI reduces manual processing time.

## 🎓 Teaching Philosophy

**"Learning by doing in the age of AI equals learning by building."**

I don't just show people what AI can do - I teach them to build AI workflows themselves. That's the only way adoption sticks.

**My methodology:**
1. **Start simple** → Claude Projects or Custom GPTs (tools people already have)
2. **Make concepts visible** → MindStudio to understand system prompts, chaining, integrations
3. **Build with real use cases** → Not tutorials. Solve actual work problems.
4. **Iterate based on feedback** → What worked? What didn't? Adjust and build again.

This is how I run AI Architect Program. This is how I work with clients.
